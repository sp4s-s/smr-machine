# Artifact Reference

## Build outputs

### `build/spsc_bench`

- generated by: `cmake --build build --target spsc_bench`
- purpose: low-level SPSC throughput and latency benchmark
- use when: validating queue performance under controlled load

### `build/command_stress`

- generated by: `cmake --build build --target command_stress`
- purpose: mixed market-style order stress with failures and replay bundles
- use when: validating live order flow, latency, failure handling, and replay capture

### `build/generate_scenario`

- generated by: `cmake --build build --target generate_scenario`
- purpose: generate deterministic `.pr42` scenarios from the C++ side

### `_build/default/ocaml/bin/deterministic_cli.exe`

- generated by: `dune build`
- purpose: deterministic replay CLI
- recommended entrypoint: `bash scripts/deterministic_cli.sh ...`

## `spsc_bench` artifacts

### `build/spsc_bench.timeline.csv`

- generated by: `./build/spsc_bench --timeline-csv ...`
- columns: `elapsed_ms,produced,consumed,interval_mps`
- use for: plotting throughput drift and backlog behavior over time

### `build/spsc_bench.latency_histogram.csv`

- generated by: `./build/spsc_bench --latency-histogram-csv ...`
- columns: latency bucket start/end and sample count
- use for: percentile and distribution charts

### `build/spsc_bench.trace_event.json`

- generated by: `./build/spsc_bench --trace-event-json ...`
- use for: portable timeline trace of the benchmark lanes and stress threads
- upload/view in:
  - Perfetto UI
  - Chrome trace viewers
  - Magic Trace only as a best-effort portable JSON import path; compatibility is not guaranteed

### macOS Instruments `.trace`

- generated by:

```bash
./scripts/profile_bench.sh
```

- use for: sampled call stacks and hotspot analysis in macOS Instruments
- open with: Instruments / `xctrace`
- do not upload to Perfetto or Magic Trace

### `build/flame/spsc_bench.samply.json.gz`

- generated by:

```bash
./scripts/profile_bench.sh
```

- use for: sampled stack flamegraphs and thread timelines in Firefox Profiler
- open with:
  - `samply load build/flame/spsc_bench.samply.json.gz`
  - [flamegraph.com](https://flamegraph.com/)
- do not treat as a native Magic Trace capture

### `build/spsc_bench.sample.txt`

- generated by:

```bash
./build/spsc_bench --messages 20000000 --capacity 1024 &
sample $! 3 -file build/spsc_bench.sample.txt
```

- use for: lightweight macOS stack sampling
- open with: any text editor
- do not upload to Perfetto, Magic Trace, or Instruments

## `command_stress` artifacts

The managed live runner writes these into the artifact directory you pass with `--artifact-dir`. The default examples in this repo use `build/trading_terminal/`.

### `build/trading_terminal/status.json`

- generated by: `lua scripts/run_trading_stress.lua --artifact-dir build/trading_terminal ...`
- purpose: current runner state plus final summary pointers
- used by: `scripts/order_terminal.lua` and the tmux status pane
- compatibility note: this file shape is preserved for migration from the deprecated Python producer

### `build/trading_terminal/run_history.jsonl`

- generated by: `scripts/run_trading_stress.lua`
- purpose: append-only history of completed managed runs
- used by: `scripts/order_terminal.lua`
- compatibility note: retains the prior JSONL shape for migration

### `build/trading_terminal/stress.stdout.log`

- generated by: `scripts/run_trading_stress.lua`
- purpose: live stdout capture of `command_stress`, including `progress_*` lines and final metrics
- use for: tailing live counters and post-run inspection

### `build/trading_terminal/stress.stderr.log`

- generated by: `scripts/run_trading_stress.lua`
- purpose: live stderr capture of `command_stress`

### `build/trading_terminal/raw_events.jsonl`

- generated by: `./build/command_stress --raw-events-jsonl ...`
- purpose: live order-by-order event stream
- used by: `scripts/order_terminal.lua`
- contents include:
  - order id
  - asset
  - trader
  - side
  - qty
  - px
  - remaining qty
  - lane
  - latency
  - failure/drop reason
- migration note: this is the primary canonical input for the Lua monitor

### `build/trading_terminal/summary.json`

- generated by: `./build/command_stress --summary-json ...`
- purpose: structured run summary
- use for:
  - throughput
  - latency percentiles
  - failure counts
  - jitter
  - dropped-after-threshold counts

### `build/trading_terminal/timeline.csv`

- generated by: `./build/command_stress --timeline-csv ...`
- columns:
  - `elapsed_ms`
  - `produced`
  - `consumed`
  - `succeeded`
  - `failed`
  - `backlog`
  - `interval_mps`
  - `moving_avg_mps`
- use for: charts of workload velocity, backlog, and failure accumulation

### `build/trading_terminal/trace_event.json`

- generated by: `./build/command_stress --trace-event-json ...`
- use for: portable event trace of producers, consumers, stress threads, counters, and failures
- upload/view in:
  - Perfetto UI
  - Chrome trace viewers
  - Magic Trace only as a best-effort portable JSON import path; compatibility is not guaranteed

### `build/trading_terminal/replays/failure_*.pr42`

- generated by: `./build/command_stress --replay-dir ...`
- purpose: replayable windows around the first failures
- use with:

```bash
bash scripts/deterministic_cli.sh replay build/trading_terminal/replays/failure_1_lane_0.pr42 --checkpoint 4 --rollback-seq 12
```

## Which files go where

### Upload to Perfetto

- `build/spsc_bench.trace_event.json`
- `build/trading_terminal/trace_event.json`

### Flamegraph Viewers

- `build/flame/spsc_bench.samply.json.gz` -> Firefox Profiler via `samply load`
- `build/flame/spsc_bench.samply.json.gz` -> [flamegraph.com](https://flamegraph.com/)

### Magic Trace

- native supported input: `trace.fxt.gz` produced by `magic-trace` on a supported Linux/Intel host
- best-effort portable file from this repo: `build/spsc_bench.trace_event.json`
- best-effort portable file from this repo: `build/trading_terminal/trace_event.json`

The repo-generated JSON files are timeline traces, not native Magic Trace captures.

### Open with macOS Instruments

- `build/spsc_bench.trace`
- any other `.trace` bundle produced by `xctrace record`

### Open locally as text or data

- `status.json`
- `run_history.jsonl`
- `stress.stdout.log`
- `stress.stderr.log`
- `raw_events.jsonl`
- `summary.json`
- `timeline.csv`
- `latency_histogram.csv`
- `sample.txt`
- `failure_*.pr42`

## Reset behavior

When you start a new managed run through `scripts/run_trading_stress.lua` or `scripts/trading_terminal_tmux.sh`, these files are cleared first by default:

- `status.json`
- `run_history.jsonl`
- `summary.json`
- `timeline.csv`
- `trace_event.json`
- `raw_events.jsonl`
- `stress.stdout.log`
- `stress.stderr.log`
- `replays/`

This keeps the live terminal and tmux panes tied to the current run instead of mixing old and new data.

Deprecation note:

- the current manager/monitor implementation for these files is Python-based and deprecated
- preserve the artifact contract while replacing the monitor/runner in Lua
